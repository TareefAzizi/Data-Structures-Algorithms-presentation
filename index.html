Leetcode 84: Largest Rectangle in Histogram

This Question is to solve the problem of finding the largest rectangular area in a given histogram. The histogram is represented by a series of bars, where the height of each bar corresponds to a given value, and the width of the bar is assumed to be 1. The task is to find the area of the largest rectangle that can be formed by any set of adjacent bars in the histogram. For example to get the area of the first bar, the height is two and the width is 1. So, width = 2 X 1 = 2.

Time and Space Complexity Analysis

Time complexity refers to the amount of time an algorithm takes to complete as the size of the input data increases. It is usually denoted by the big O notation (O(n)), where n is the size of the input. The following are some common time complexities:

  • O(1) - Constant Time: An algorithm that takes the same amount of time to complete regardless of the input size. Examples include accessing an array element or performing a single arithmetic operation.
  • O(log n) - Logarithmic Time: An algorithm that takes time proportional to the logarithm of the input size. Examples include binary search or searching in a balanced binary search tree.
  • O(n) - Linear Time: An algorithm that takes time proportional to the input size. Examples include linear search or traversing an array.
  • O(n log n) - Linearithmic Time: An algorithm that takes time proportional to n times the logarithm of n. Examples include sorting algorithms like merge sort and quicksort.
  • O(n^2) - Quadratic Time: An algorithm that takes time proportional to the square of the input size. Examples include bubble sort and selection sort.
  • O(2^n) - Exponential Time: An algorithm that takes time proportional to 2 raised to the power of the input size. Examples include the brute force solution to the traveling salesman problem.

Space complexity refers to the amount of memory space an algorithm takes up as the size of the input data increases. It is also denoted by the big O notation (O(n)), where n is the size of the input. The following are some common space complexities:

  • O(1) - Constant Space: An algorithm that takes up a fixed amount of memory regardless of the input size. Examples include swapping two variables or reversing an array in place.
  • O(n) - Linear Space: An algorithm that takes up space proportional to the input size. Examples include creating a new array to store the result of an operation on the input array.
  • O(n^2) - Quadratic Space: An algorithm that takes up space proportional to the square of the input size. Examples include creating a 2D array to store the result of a matrix multiplication.

In the worst case, when the sliding window covers the entire array, the size of the queue q can be equal to the size of the window k, which contributes to the space complexity. The Monotonic Deque. Is a Deque. What is a Deque? Deque or Double Ended Queue is a type of queue in which insertion and removal of elements can either be performed from the front or the rear. Thus, it does not follow FIFO rule (First In First Out). For example like a Stack. A Monotonic Deque is the elements in the deque from head to tail are in decreasing order.

Method: Monotonic Stack

Time Complexity: O(n) where n is the number of elements in the input array nums.

Space Complexity :O(n) because it uses a stack data structure that can hold at most n elements in the worst case, where n is the length of the heights array.

The Monotonic Stack. Is a Stack. What is a Stack? A stack is a linear data structure that follows the principle of Last In First Out (LIFO). This means the last element inserted inside the stack is removed first. Monotonic stack is like a regular stack with one key distinction in the push operation: Before we push a new element onto the stack, we first check if adding it breaks the monotonic condition. If it does, then we pop the top element off the stack until pushing the new element no longer breaks the monotonic condition.

Leetcode 23. Merge k Sorted Lists

First Solution:

Method: Merge Sort

Time Complexity: O(N log k), where N is the total number of nodes in all linked lists and k is the number of linked lists

Space Complexity :O(k), where k is the number of linked lists. This is because the code creates a new node for each merged node, which adds up to a total of k nodes.

Merge sort is a divide-and-conquer algorithm that sorts an array by dividing it into two halves, sorting each half recursively, and then merging the two sorted halves into one sorted array.

Code Explanation:

We make a function called mergeKlists that takes in `lists` as its parameter.

A variable called `previous` is assigned to a value which is null. This is to keep tracked of the merged linked list

Then a for loop is declared to loop through the merged linked lists.

For each linked lists we call the function `mergeTwolists` passing in the previous linked list and the curent linked list as the arguement

After merging all the linked lists we return previous.

Now we take the `mergeTwoLists` function that takes in two linked lists. `list1` and `list2,

Then we create ListNode with a value of 0. This will act as a sentinel node, which is a common technique used when working with linked lists. A sentinel tracks the of the front of the list and the tail trackes the last values.

After that we declare a while loop to iterate through both lists and adding nodes to the new linked lists in a correct order. This while loop runs as long as both list1 and list2 have nodes that haven't been added to the new list yet.

Inside the loop, we check which of the two lists has the smaller first value. We do this by comparing the .val property of the first node in each list.

Then the first `If statement` checks if If list1 has the smaller value (or if the two values are equal), we add the first node from list1 to the new list and advance the pointer for list1 to the next node.

Then the second `else statement` returns if list2 has the smaller value, we add the first node from list2 to the new list and advance the pointer for list2 to the next node.

After adding a node to the new list, we advance the pointer for tail to the next node in the new list.

Once the loop is finished, we add any remaining nodes from list1 or list2 to the end of the new list.

Then we return the new linked list
